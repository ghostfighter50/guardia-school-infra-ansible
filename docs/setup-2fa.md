# Two-Factor Authentication Setup

Guide to TOTP-based two-factor authentication for SSH access.

---

## Overview

All user accounts (except the `ansible` service account) require a time-based one-time
password (TOTP) in addition to an SSH key to authenticate. TOTP secrets are generated by
Ansible at deploy time, stored in Vault, and must be added to an authenticator app by each user.

---

## Prerequisites

| Requirement | Notes |
|-------------|-------|
| SSH hardening applied | Run setup-ssh.md steps first |
| Vault deployed and unsealed | Secrets are retrieved from Vault |
| Authenticator app | Google Authenticator, Authy, andOTP, or any TOTP-compatible app |

---

## How TOTP Works

1. The `ssh_hardening` role generates a TOTP secret using `google-authenticator` at deploy time
2. The secret is stored in Vault at `secret/hosts/<hostname>/admin_2fa`
3. The managed user role generates a separate TOTP for the `deploy` user
4. When logging in via SSH, the user provides their SSH key first, then a 6-digit code
5. The code is valid for 30 seconds and is generated by an authenticator app

---

## Step 1: Retrieve Your TOTP Secret

### Admin user TOTP

```bash
export VAULT_ADDR=http://127.0.0.1:8200

vault kv get secret/hosts/<hostname>/admin_2fa
```

Output fields:

| Field | Description |
|-------|-------------|
| `secret` | The TOTP base32 secret key |
| `scratch_codes` | Emergency one-time backup codes |
| `qr_url` | otpauth:// URL for QR code generation |

### Managed user (deploy) TOTP

```bash
vault kv get secret/hosts/<hostname>/2fa
```

---

## Step 2: Add Secret to Authenticator App

### Option A: Scan QR code (recommended)

Generate a QR code from the `qr_url` field:

```bash
# Install qrencode if needed
sudo apt install -y qrencode

# Generate QR code in terminal
echo "<qr_url_value>" | qrencode -t ANSIUTF8
```

Scan the QR code with your authenticator app.

### Option B: Manual entry

In your authenticator app, choose "Enter key manually":

| Field | Value |
|-------|-------|
| Account name | admin@<hostname> |
| Key | The value from the `secret` field |
| Type | Time-based |

---

## Step 3: Test Authentication

```bash
# Attempt SSH login - you will be prompted for a TOTP code after key auth
ssh -p 2222 admin@<target-ip>

# Expected flow:
# 1. SSH key checked automatically
# 2. Prompt: "Verification code:"
# 3. Enter 6-digit code from your app
# 4. Login succeeds
```

## Scratch Codes

Scratch codes are single-use emergency codes stored in Vault. Use them only if you have
lost access to your authenticator app.

```bash
# Retrieve scratch codes
vault kv get -field=scratch_codes secret/hosts/<hostname>/admin_2fa
```

Each code can only be used once. After use, regenerate TOTP by re-running:

```bash
ansible-playbook playbooks/02_harden.yml --tags ssh --limit <hostname>
```

Note: This generates a new TOTP secret. The old one becomes invalid.

## Verification

```bash
# Confirm TOTP file exists on target
ansible linux_targets -m command -a "stat /home/admin/.google_authenticator" -b

# Confirm PAM is configured for TOTP
ansible linux_targets -m command -a "grep pam_google_authenticator /etc/pam.d/sshd" -b

# Confirm secret is in Vault
vault kv list secret/hosts/<hostname>/
```
